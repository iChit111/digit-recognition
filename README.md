# 🥷 Neural Ninjas — Handwritten Digit Recognition

A beginner-friendly deep learning project that trains a neural network to recognise handwritten digits (0–9) using the MNIST dataset and TensorFlow/Keras.

Built by **Team Neural Ninjas** as a group project at **PCCOE Pune**.

---

## 📌 What This Project Does

- Loads and explores the MNIST dataset (70,000 handwritten digit images)
- Trains a fully-connected neural network to classify digits 0–9
- Evaluates the model's performance with accuracy scores and a confusion matrix
- Runs an interactive demo that predicts a digit and shows a confidence chart

The final model achieves **~97.5% accuracy** on the test set.

## ⚙️ Setup Instructions

### 1. Clone the repository
```bash
git clone https://github.com/<your-username>/digit-recognition.git
cd digit-recognition
```

### 2. Make sure you have Python 3.9–3.11 installed
```bash
python --version
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

This installs TensorFlow, Keras, NumPy, Matplotlib, scikit-learn, and Jupyter.

---

## 🚀 How to Run (in order)

The scripts must be run **in this exact order** from the project root directory.

### Step 1 — Load and explore the data
```bash
python src/data_loader.py
```
- Downloads the MNIST dataset automatically (first run only)
- Shows a 5×5 grid of sample images
- Shows a bar chart of how many images exist per digit
- Preprocesses images (normalise + flatten)

### Step 2 — Train the model
```bash
python src/model.py
```
- Builds a 3-layer neural network (784 → 128 → 64 → 10)
- Trains for 10 epochs on 54,000 training images
- Plots and saves training loss/accuracy curves
- Saves the trained model to `results/model.h5`

### Step 3 — Evaluate the model
```bash
python src/evaluate.py
```
- Loads `results/model.h5`
- Reports test loss and accuracy on 10,000 unseen images
- Generates a confusion matrix heatmap
- Shows the first 15 images the model got wrong

### Step 4 — Run the demo
```bash
python src/demo.py                        # Random test image
python src/demo.py --index 42             # Specific image (0–9999)
python src/demo.py --random --count 5     # 5 random predictions in a row
```
- Loads `results/model.h5`
- Displays the digit image alongside a confidence bar chart
- Saves each prediction to `results/demo_prediction_*.png`

## 🧠 Model Architecture

```
Input Layer   →  784 units  (one per pixel of the 28×28 image)
Hidden Layer 1 → 128 units  (ReLU activation)
Hidden Layer 2 →  64 units  (ReLU activation)
Output Layer  →  10 units   (Softmax activation — one per digit 0–9)
```

| Setting          | Value                          |
|------------------|-------------------------------|
| Optimizer        | Adam                          |
| Loss function    | Sparse Categorical Crossentropy |
| Epochs           | 10                            |
| Batch size       | 32                            |
| Validation split | 10%                           |
| Test accuracy    | ~97.5%                        |
